# AI校园学习助手系统 — 项目设计文档（Java版）

## 一、项目背景

### 1.1 背景介绍

随着人工智能技术的快速发展，AI已经深度融入教育领域。大学生在日常学习中面临诸多挑战：课程知识点理解困难、学习计划缺乏科学性、编程代码调试效率低下、论文写作思路不清等。传统的在线学习工具功能单一，无法为学生提供智能化的综合学习辅助。

本项目旨在开发一个基于Spring Boot框架的AI校园学习助手系统，集成DeepSeek大语言模型API，为学生提供智能问答、作文生成、学习计划制定、代码解释等功能，帮助学生提高学习效率，实现个性化学习。

### 1.2 项目目标

- 构建一个功能完整的Java Web应用系统，涵盖用户管理、AI对话、内容生成等核心功能
- 实现标准的MVC三层架构，代码结构清晰、可维护性强
- 集成AI大模型能力，提供智能化的学习辅助服务
- 提供管理员后台，支持系统运维和数据管理
- 完整的项目文档，可直接用于课程综合实训报告

### 1.3 技术选型依据

| 技术 | 选择理由 |
|------|----------|
| Java 17 | 企业级开发主流语言，稳定性强，高校教学常用 |
| Spring Boot 3.2 | 最流行的Java Web框架，自动配置，快速开发 |
| MyBatis-Plus 3.5 | 增强版MyBatis，简化CRUD操作，提高开发效率 |
| MySQL 8 | 企业级关系型数据库，性能稳定，高校教学常用 |
| Thymeleaf | Spring Boot官方推荐的模板引擎，服务端渲染 |
| Bootstrap 5 | 响应式前端框架，快速构建美观界面 |
| JWT | 无状态认证方案，适合前后端分离架构 |
| DeepSeek API | 国产大语言模型，性价比高，接口兼容OpenAI格式 |

### 3.3 技术架构说明

**表现层（Templates + Static）**
- 使用Thymeleaf模板引擎渲染HTML
- Bootstrap 5实现响应式布局
- Axios处理异步请求
- marked.js渲染Markdown
- highlight.js实现代码高亮

**控制层（Controller）**
- AuthController: 用户认证相关
- ChatController: AI聊天相关
- ArticleController: 作文生成相关
- PlanController: 学习计划相关
- CodeController: 代码解释相关
- AdminController: 管理员后台
- UserController: 用户中心

**业务层（Service）**
- AIService: AI接口抽象层，支持DeepSeek/OpenAI/Ollama切换
- UserService: 用户业务逻辑
- ChatService: 聊天业务逻辑
- ArticleService: 文章业务逻辑
- PlanService: 学习计划业务逻辑
- CodeService: 代码解释业务逻辑

**持久层（Mapper）**
- 使用MyBatis-Plus简化CRUD操作
- 支持自定义SQL的XML映射文件
- 模型与数据库表一一对应
